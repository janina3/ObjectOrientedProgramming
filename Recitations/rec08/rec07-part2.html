PART TWO

We are writing code to model a little world that has an orchestra, musicians and a Musical Instrument Lending Library.

We've gotten working classes for the MIL2 (Musical Instrument Lending Library), the Musican, and the Instrument hierarchy.

Now it's time to use them.

We'll be modeling a little world in which things happen.

The classes are created to allow a programmer to write code that models the activities in the world.
We'll call this a scenario.

Here's a possible scenario in this (slightly) expanded musical world:

bob joins the orchestra without an instrument.
the orchestra performs
sue gets an instrument from the MIL2 and joins the orchestra.
ralph gets an instrument from the MIL2.
mary gets an instrument from the MIL2 and joins the orchestra.
ralph returns his instrument to the MIL2.
jody gets an instrument from the MIL2 and joins the orchestra.
morgan gets an instrument from the MIL2
the orchestra performs.
ralph joins the orchestra.
the orchestra performs.
bob gets an instrument from the MIL2
ralph gets an instrument from the MIL2
the orchestra performs.
morgan joins the orchestra.
the orchestra performs.

A programmer would use objects to model this world by calling methods.

There will most always be an "initialization phase" in which the objects that will be used in the scenario will be created and things that need to be preset are (like there should be a MIL2 and someone needs to put some instruments in it before the scenario starts). This is done by the modeling programmer before the code for doing the scenario.

There might be some clean up code after the scenario. In this recitation there isn't.

What are the additional clases that will need to be created for this scenario?
Only orchestra. As noted, we already have the others.

Q: What additional functionality needs to be added to some of our classes?
A: The way to play an instrument "normally." All instruments must have this ability.

There are two ways to play an instrument: the normal playing way that musicians use when performing in the orchestra and the making sound way that the MIL2 uses each morning to test the instruments and that musicians used in part one. (The MIL2 could play its instruments in the normal way but it just doesn't.)

  • For Normal PLAYing:
    • For trumpets the output should also be:
      "Toot"
    • For trombones the output should also be:
      "Blat"
    • For violins the output should be:
      "Screech"
    • For cellos the output should be:
      "Squawk"
    • For drums the output should be:
      "Boom"
    • For cymbals the output should be:
      "Crash"

How to enforce that every instrument must have a play method?

How would you model "the orchestra?" - an object that "contains" all the musicians.
The orchestra has a maximum size of 25. Any musician can join the orchestra but she joins for life! It also doesn't matter if a musician doesn't have an instrument when they are in the orchestra. When the orchestra performs, any musician who doesn't have an instrument just doesn't play their instruments (or should that be stated: they don't play the instruments that they don't have?!). A musician can go to the MIL2 and get an instrument and then when the orchestra plays, they play. When the orchestra performs, it just has each of its musicians one at a time play their instrument in the normal way. The order in which the musicians perform doesn't matter as long as all musicians in the orchestra are told to play. You already have a working container for instrument pointers so there shouldn't be too much work to modify that code to instead contain Musician*'s.
