PART ONE (do PART TWO after you have finished PART ONE)

The group will discuss and solve the design phase for both PARTs before anyone starts writing code.

We are writing code to model for a Musical Instrument Lending Library and some musicians.

Our MIL2 (Musical Instrument Lending Library) provides the musicians in this land with instruments. It's sort of like an old-style Communist system in that any musician can go to the MIL2 and get an instrument for no charge (but they have no choice in what kind of instrument the get!). Musicians will return their instrument to the MIL2 at some point.

  • When the MIL2 receives an instrument,
    • it first tests the instrument by making a sound on it (see the section "Making Sound" below)
    • then places that instrument in its inventory at the first available place in the inventory.
      (it doesn't actually place the instrument itself, of course. The instrument exists outside the MIL2 and we don't want to make a copy to be stored - there should be only one instance of that instrument. A pointer to the instrument to be stored would be added to the inventory.)
    • The MIL2's inventory consists of instruments.
  • When the MIL2 loans out an instrument,
    • it returns the next available instrument (starting at index 0).
      What if there aren't any left? Hm, if we are returning instruments, this is difficult. Instead, we could return something different. What? The address of the instrument. How does this help? If we don't have any instruments we can return NULL.
    • it no longer has that instrument in its inventory
      (but the instrument itself still exists, of course)

Each morning before the MIL2 opens its doors, it tests all the instruments in its inventory by making sounds on them.
This is modeled by writing output (shown below).

At the start of the modeling code, the MIL2 is filled with instruments by the modeling programmer - see tester code (link below).

Instruments

In the world we are modeling, there are only brass instruments, string instruments and percussion instruments.

Brass

Each brass instrument has a mouthpiece which has a size (an unsigned). There are three different kinds of brass instruments based on the size of their mouthpieces: Big, Medium, and Small which have mouthpiece sizes of 12, 9 and 4 respectively. There are no other mouthpiece sizes in this world. Making a sound is done by blowing on the mouthpiece. We'll model this making of sound by writing output on the screen (shown below). Another classification of brass instruments is that there are trumpets and trombones. They differ from each other in the sounds they make when they are played but this will dealt with in part two of the recitation. So there will really be nothing except the constructors for trumpets and trombones in part one.

String

String instruments in this world have only one string and it has a pitch (an unsigned) which is set when the string instrument is created and never changes. For both, making a sound is done by bowing the string to produce the pitch. We'll model this making of sound by writing output on the screen (shown below). The two kinds of stringed instruments we have in this world are violins and cellos. They differ from each other in the sounds they make when played but this will dealt with in part two of the recitation. So there will really be nothing except the constructors for violins and cellos in part one.

Percussion

Percussions instruments in this world, like most all musical worlds are hardly considered real instruments. They have no data members. For both, making a sound is done by hitting them.The two kinds of percussion instruments we have are drums and cymbals which make different sounds when played but this will dealt with in part two of the recitation. So there will really be nothing except the constructors for drums and cymbals in part one.

Making Sound

All instruments must be able to make sounds.

Specifically we want the output to the screen for making a sound to be like this:

  • To MAKE SOUND:
    • For all brass instruments the output should also be:
      "To make a sound... blow on a mouthpiece of size 9"
      (9 would be the size of the mouthpiece that this particular woodwind instrument has)
    • For all string instruments the output should be:
      "To make a sound... bow a string with pitch 567"
      (567 would be the pitch that this particular stringed instrument has)
    • For all percussion instruments the output should be:
      "To make a sound... hit me!"

Recall that in designing with inheritance, you try to move data members and methods "up" (towards the base) as far as possible.

The object we are most focused on in this recitation is the MIL2. Consider the virtual mechanism when implementing this class' methods. (Will it have any virtual methods itself?) And abstract classes as ways to "enforce" that derived classes are consistent.

Into the Future!

For the future (and for those without the requisite musical knowledge) :

In addition to drums and cymbals, tambour, tambourine, grandeCasse, triangle. bongo, conga, timbale, castanets, maracas, guiro, as well as xylophone, orchestra bells, chimes, marimba, vibraphone (vibes), crotales and amgloken are in the percussion family.

Trumpet, flugelhorn, bugle, trombone, French horn, euphonium, tuba, cor de Chasses, sackbutt, zink, and sousaphone are also in the brass family.

Other stringed instruments are viola, contrabass violin (bass), guitar, lute, banjo, ukelele, balalaika, oud, sitar, zither, and kora.

Flute, saxophone, clarinet, oboe, bassoon, shawm, cor Anglais, krummhorn and rackett are in the woodwind family which is a kind of wind instrument family that is completely different from brass instruments - but they are still instruments.

You should write your code so that in the future, if any of these kinds of instruments are needed, your C++ program would be ready to handle them with no (or almost no) modifications - because of the way you wrote your code for the classes using inheritance.
